 I will happily send people a locally-written program that creates a
/u directory full of symbolic links to people's home directories. Right
now it uses the password file to figure out where the symbolic links go,
but it should be fairly easy to make it use another source of information.

 Functions and shell features and the like are the wrong way; /u is the
right way. Having it in the filesystem makes all sorts of things just
*work* that having it in shells prevents.
