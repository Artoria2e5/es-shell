I agree that if you specify an fd used by es (e.g., the history file) then
es should move it out of the way.  More sugar (or a whole nother type such
as nutrasweet) is needed to make newfd the easy thing to use.

I don't think "exec" should mean that with-open-file thing; it should set
not push -- whoever said es should come back to its old stdin after input
is exhausted was wrong.

However, I really REALLY like the idea of separating out es's command input
stream from the stdin of processes, but I'm not sure it doesn't have other
impacts and would need to think about it (more realistically:  have Paul and
Byron think about it :-).
