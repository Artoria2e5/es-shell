The "decompression hack" is supported in SR10.4.  If you compress a file 
and give it a object type of "compress" it will be expanded for you when 
opened.  For those not in the know, the "object type" is a characteristis
of entities in Apollos file system.  It bears no relation to the usual
Unix-oid meaning of "type" being represented as, say ".Z".  Thus, no
naming conventions have to be fiddled with.
