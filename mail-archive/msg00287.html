As the one who started the whole mess, maybe I can gain some atonement, so
here's an idea on a replacement for <>{}.  How about

	<-{}

It has been pointed out to me that this presents only the minor problem
of using filenames that begin with `-' right after the < for i/o
redirections, but I don't see that as a big deal (We can't use a file named
'<' either, or << doesn't work... :-).  It gives us back the "3 operators for
3 operations" that we had before, which I think is a major win. It also allows
<-$&primitive which <> did in initial.es.  Hmm, I think that doesn't look
half bad.

(Is dws on the list?  What kind of smiley is <-{} ?)

Yays, nays, consensus or lack thereof, to the list please.

Thanks,

Arnold
